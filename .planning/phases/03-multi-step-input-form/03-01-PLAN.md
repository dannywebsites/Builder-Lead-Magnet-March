From src/lib/calculator/schemas.ts:
```typescript
export const EntityTypeSchema = z.enum(["limited_company", "sole_trader"]);
export const VatRateSchema = z.enum(["0", "0.135", "0.20", "0.23"]);
export const CurrencySchema = z.enum(["GBP", "EUR"]);
export const CalculatorInputSchema = z.object({ ... });
export type ValidatedCalculatorInput = z.infer<typeof CalculatorInputSchema>;
```

CRITICAL: VatRateSchema uses STRING enums ("0", "0.135", "0.20", "0.23") but CalculatorInput.vatRate uses NUMERIC VatRateValue (0, 0.135, 0.2, 0.23). The transform layer bridges this gap.

CRITICAL: directCostPct in the schema is 0.0-0.8 (decimal), but the UI displays 0-80 (percentage). The form uses a separate field `directCostPctDisplay` (0-80) that gets divided by 100 in the transform.
